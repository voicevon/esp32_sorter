# PS100 伺服驱动器 RS485 控制技术指南 (Project Wiki)

本文档记录了在 ESP32 Sorter 项目中实现普菲德 (PFDE) PS100 伺服驱动器通过 RS485 进行 0-3000 RPM 闭环速度控制的技术细节。

## 1. 硬件连接 (Physical Layer)

### 1.1 ESP32 侧引脚
*   **UART**: Serial1 (Pins: TX 17, RX 16)
*   **EN**: PIN 5 (RS485 转换模块的 DE/RE 控制位)
*   **Baud**: 9600 bps (8N1)

### 1.2 伺服侧 RJ45 接口 (CN3/CN4)
| 针脚 (Pin) | 定义 | 说明 |
| :--- | :--- | :--- |
| 4 | RS485- (B) | 与 ESP32 模块 B 连接 |
| 5 | RS485+ (A) | 与 ESP32 模块 A 连接 |
| 7 | GND | 共地 (建议连接以降低干扰) |

## 2. 驱动器面板手动基础配置

**注意**: PA4 模式修改必须在驱动器未使能状态下进行，修改后**必须断电重启**。

| 参数编号 (PA) | 设定值 | 描述 |
| :--- | :--- | :--- |
| **PA 0** | 315 | 用户密码 (用于解锁高级参数) |
| **PA 4** | 1 | **控制模式 (Servo Drive)**: 1=速度控制 |
| **PA 22** | 1 | **速度指令源**: 1=内部速度/Modbus |
| **PA 53** | 1 | **强制使能**: 1=启动即转 |
| **PA 71** | 1 | **站号**: Modbus Slave ID |
| **PA 72** | 96 | **波特率**: 9600 bps |
| **PA 73** | 3 | **通讯协议**: 8N1 (RTU 模式) |

### 面板操作流程 (四键面板: 左、上、下、SET):
1.  连续点击 **[左]** 键切换至 `PA-00` 菜单。
2.  进入参数后，按 **[上/下]** 寻找具体 PA 编号。
3.  点击 **[SET]** 进行值编辑。
4.  **长按 [SET] 3秒以上** 看到 `done` 字样才算保存。

## 3. Modbus 协议详情 (Protocol)

### 3.1 关键寄存器地址 (Hex)
*   **PA 24 (目标转速)**: `0x0018` (写入 0-3000)
*   **PA 53 (强制使能)**: `0x0035` (写入 1 使能，0 停止)
*   **PA 4 (控制模式)**: `0x0004`
*   **PA 0 (机器码/心跳)**: `0x0000` (回读用于检查链路)

### 3.2 控制逻辑
*   **步进调整**: 50 RPM (由编码器旋转脉冲触发)
*   **启动同步**: 系统启动时自动发送 PA4=1, PA22=1, PA53=1 以确保状态同步。

## 4. 代码实现细节 (Logic)

### 4.1 发送指令 (Write)
```cpp
void writeModbusRegister(uint16_t regAddr, uint16_t regValue) {
    uint8_t frame[8] = {
        0x01,                   // Slave ID
        0x06,                   // Function Code 0x06
        regAddr >> 8, regAddr,  // Address
        regValue >> 8, regValue,// Value
        0x00, 0x00              // CRC (placeholder)
    };
    // 计算并填入 CRC...
    Serial1.write(frame, 8);
}
```

### 4.2 诊断机制
*   **心跳检测**: 诊断模式下循环读取 PA 寄存器，若连续 2 秒无响应则报 "No Link"。
*   **原始数据**: 开启 `[Modbus TX/RX]` 串口输出，核对报文完整性。

## 5. 故障预判与解决 (Troubleshooting)

*   **通讯 OK 但电机不转**:
    *   检查 `PA 4` 读值。若非 1，需手动在面板设置并重启。
    *   检查 `PA 53`。若为 0，驱动器不会锁定电流。
*   **控制失效**: 确认所使用的控制模式是否支持 Modbus（见下表）。

## 6. 控制模式支持矩阵 (Mode Support Matrix)

| 控制模式 (PA4) | 描述 | Modbus (RS485) 支持 | 脉冲/模拟量支持 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **0** | 位置模式 | **不支持** | 支持 (脉冲+方向) | 必须使用外部脉冲串控制 |
| **1** | 速度模式 | **支持** (PA22=1) | 支持 (模拟量) | 推荐在此模式下进行传送带控制 |
| **2** | 转矩模式 | **支持** | 支持 (模拟量) | 可通过 Modbus 限制最大转矩 |
| **6** | 内部 PR 模式 | 部分支持 | - | 本项目不使用 |

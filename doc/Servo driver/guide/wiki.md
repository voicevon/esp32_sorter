# PS100 伺服控制器 - RS485 控制技术 Wiki

本 Wiki 汇总了普菲德 (PFDE) PS100 伺服驱动器通过 Modbus RTU 协议进行深度控制的技术细节，重点解决 **模式切换**、**实时调速** 与 **转矩控制** 逻辑。

## 1. 通讯基础配置
- **波特率**: 9600 bps (PA72)
- **站号**: 1 (PA71)
- **协议**: 8N1 (PA73=3, 对应 RTU 模式)
- **物理连接**: ESP32 Serial1 (TX17, RX16) -> RS485 模块 -> 伺服 RJ45 (4:B, 5:A, 7:GND)

## 2. 模式切换 (PA4)
修改 PA4 后，建议执行 **“存盘+复位”** 序列以确保硬件层驱动切换。
- **地址**: `0x0004` (ROM), `0x0084` (RAM)
- **存盘指令**: `PA0=2` (地址 `0x0000`)
- **复位指令**: `PA60=1` (地址 `0x003C`, 官方 FA-60 复位)

### 常用模式值:
- `0`: 位置模式 (不推荐通过 Modbus, 响应慢)
- `1`: 速度模式 (适合输送线)
- `2`: 转矩模式 (适合张力控制或分拣受限控制)

## 3. 速度控制 (Velocity Mode)
1. **指令源 (PA22)**: 必须设为 **`2` (通讯速度指令)**。
2. **给定值 (PA24)**: 指向地址 **`0x0018`** (ROM) 或 **`0x0098`** (RAM)。
3. **使能 (PA53)**: 设为 `1`。

## 4. 转矩控制 (Torque Mode)
1. **指令源 (PA32)**: 必须设为 **`2` (通讯转矩指令)** 或依据特定固件设为 `1`。建议首选 `0x0020=2`。
2. **给定值**:
   - 方式 A: **PA24 (0x0018)** - 部分固件共享地址。
   - 方式 B: **PA64 (0x0040)** - 专项转矩给定地址 (推荐尝试)。
3. **安全限速 (PA31/PA50)**: 在转矩模式下必须设置 `PA31`（如 500 RPM），否则电机在空载时会无限加速导致损毁。

## 5. 寄存器速查表 (Hex)

| 参数/寄存器 | 地址 | 描述 | 暂存地址 (+0x80) |
| :--- | :--- | :--- | :--- |
| **控制字** | `0x2000` | 控制使能 (Bit0)、报警复位 (Bit1) | - |
| **状态字** | `0x2001` | 就绪、使能、报警状态位 | - |
| **实时转速** | `0x1000` | 电机当前实际 RPM | - |
| **实时转矩** | `0x1007` | 电机当前负载 % | - |
| **脉冲位置** | `0x1001-2` | 32位累积脉冲 (CDAB 格式) | - |
| **PA-22** | `0x0016` | 速度源源 (0:模拟, 1:寄存器, 2:通讯) | `0x0096` |
| **PA-32** | `0x0020` | 转矩源源 (0:模拟, 1:内部, 2:通讯) | `0x00A0` |
| **PA-24** | `0x0018` | 核心指令给定值 (速度/转矩) | `0x0098` |

## 6. 开发建议逻辑 (点火序列)
1. 写入 `PA4` 目标模式。
2. 写入 `PA22=2` 或 `PA32=2` 确立通讯霸权。
3. 执行 `PA0=2` 固化并在 `PA60=1` 后等待驱动器 Online。
4. 周期性（如 50ms）更新指令寄存器 `0x0098` 或 `0x0040`。

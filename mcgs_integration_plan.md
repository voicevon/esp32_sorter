# 昆仑通态 (MCGS) 串口屏深度集成实施计划 (V3)

本方案不仅涵盖 ESP32 的代码集成，还包含了对 MCGS 组态逻辑的“协议主导”设计。

## 1. 寄存器地址分配 (主导协议)
我们将采用 **Modbus 4xxxx (保持寄存器)** 区进行通讯。
*注：MCGS 中的地址通常从 1 开始（即对应 Modbus 通讯中的 0 地址）。*

### A. 系统状态区 (Read Only for Screen)
| Modbus 地址 | 功能说明 | 数据格式 | 备注 |
| :--- | :--- | :--- | :--- |
| `40001` | 系统模式 (System Mode) | uint16 | 0:Normal, 1:Encoder, 2:Scanner, 3:Outlet |
| `40002` | 指示 LED 状态位 (595 镜像) | uint16 | 位图格式，用于屏幕模拟指示灯 |
| `40003` | 输送带负载率 / 运行频率 | uint16 | 0-1000 (代表 0.0-100.0%) |

### B. 生产数据区 (Read Only for Screen)
| Modbus 地址 | 功能说明 | 数据格式 | 备注 |
| :--- | :--- | :--- | :--- |
| `40010` | 实时测量直径 (mm) | uint16 | 放大 100 倍。例: 2250 = 22.50mm |
| `40011` | 分拣速度 (pcs/h) | uint16 | 3600 代表 1 根/秒 |
| `40012` | 总计计数 (Total) | uint32 | 占用两个寄存器 (40012, 40013) |
| `40014` | 识别计数 (Identified) | uint32 | 占用两个寄存器 (40014, 40015) |

### C. 触控控制区 (Read Write)
| Modbus 地址 | 功能说明 | 数据格式 | 备注 |
| :--- | :--- | :--- | :--- |
| `40100` | **触控命令码 (Cmd)** | uint16 | 1:MENU, 2:OK, 3:UP, 4:DOWN, 5:BACK |
| `40101` | 参数修改索引 (Index) | uint16 | 用于标识屏幕正在修改哪个出口的阈值 |
| `40102` | 参数修改数值 (Value) | uint16 | 屏幕输入的新数值 |

## 2. ESP32 侧架构变更

### [NEW] 模块：`ModbusDisplay` (类定义)
- **基类**：`Display`
- **职责**：
    - 在 `vUITask` 中，每 50ms 向 ModbusMaster 提交一次 `asyncRead(40100)`。
    - 在 `vControlTask` 反馈数据时，通过 `asyncWrite` 更新 `40010` 等寄存器。
    - **防抖逻辑**：读取到 `40100` 非零后，立即执行动作，并异步将该地址回写为 0 (Handshake)。

### [MODIFY] 模块：`UserInterface`
- 增加一个“逻辑输入缓冲区”，允许 `ModbusDisplay` 注入按键事件。
- 这样，无论你按实体按钮还是屏幕上的虚拟按钮，调用的都是同一套菜单逻辑。

## 3. 实施路径

### 第一阶段：基础设施 (移植)
1. 复制 `ModbusMaster.h/cpp`。
2. 在 `config.h` 中配置 485 硬件：
   - `PIN_HMI_485_TX = 17`
   - `PIN_HMI_485_RX = 16`
   - `PIN_HMI_485_EN = 5` (驱动 H 桥转换)

### 第二阶段：协议定义与模拟
1. 创建 `src/user_interface/screens/hmi_registers.h`，将上述地址表硬编码。
2. 编写 `ModbusDisplay.cpp`，实现对这些地址的读写包装。

### 第三阶段：部署辅助 (针对 U 盘下载)
1. 生成一份 **《MCGS 组态点位导出表》**。
2. 在 MCGS 软件中新建一个 `Modbus RTU Master` 驱动，将 U 盘插入电脑，在组态软件里配置好这些地址即可。

## 4. 验证方案
- 使用 **Modbus Poll** 软件在电脑端模拟屏幕，观察 ESP32 是否能正确通过 RS485 读写寄存器。
